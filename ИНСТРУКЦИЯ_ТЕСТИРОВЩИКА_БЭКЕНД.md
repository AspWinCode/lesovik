## Ручное тестирование — Swagger UI

Открыть `http://localhost:8000/docs`.

### Получить токен

1. Открыть `POST /api/v1/auth/login`
2. Нажать **Try it out**
3. Вставить:
```json
{
  "email": "admin@example.com",
  "password": "YourPassword1!"
}
```
4. Нажать **Execute** → скопировать `access_token`
5. Нажать кнопку **Authorize** (замок вверху страницы)
6. Вставить токен в поле `Bearer {token}` → **Authorize**

После этого все запросы из Swagger будут авторизованы.

## Типичные сценарии проверки

### Сценарий 1: Регистрация и логин

```
POST /users   → создать пользователя
POST /auth/login  → получить токены
GET  /users/me    → убедиться, что вернул правильного пользователя
```

### Сценарий 2: CRUD приложения

```
POST /apps             → создать приложение (slug уникален)
POST /apps (тот же slug) → ожидать 409 Conflict
GET  /apps             → список содержит новое приложение
PATCH /apps/{id}       → обновить имя
POST /apps/{id}/publish → статус is_published = true
DELETE /apps/{id}      → архивировать (is_archived = true)
GET  /apps/{id}        → 404 (если нет доступа) или is_archived=true
```

### Сценарий 3: Таблица + записи

```
POST /entities          → создать таблицу "Товары"
POST /entities/{id}/fields → добавить поле "Цена" (type: decimal)
POST /records           → создать запись {Цена: 999.99}
GET  /records?sort=price:desc → список с сортировкой
GET  /records?filter=price__gt:500 → фильтр
PATCH /records/{id}     → изменить Цену
DELETE /records/{id}    → удалить
```

### Сценарий 4: Права доступа

```
Создать пользователя-редактора
Добавить его в приложение: POST /apps/{id}/members {role: "editor"}
Залогиниться как редактор
PATCH /apps/{id} → должен пройти
DELETE /apps/{id} → должен вернуть 403
```

### Сценарий 5: Rate limit

```
POST /auth/login (10 раз за минуту) → на 11-й запрос 429 Too Many Requests
```

## Коды ошибок — что ожидать

| Код | Когда |
|-----|-------|
| `200` | Успешное чтение / обновление |
| `201` | Ресурс создан |
| `204` | Успешное удаление |
| `400` | Невалидный запрос (Pydantic ошибка) |
| `401` | Не авторизован / токен истёк |
| `403` | Недостаточно прав |
| `404` | Ресурс не найден (или скрыт от пользователя) |
| `409` | Конфликт (slug уже занят, запись уже существует) |
| `422` | Ошибка валидации Pydantic (тело ответа содержит detail) |
| `423` | Edit lock захвачен другим пользователем |
| `429` | Rate limit (слишком много запросов) |
| `500` | Внутренняя ошибка сервера |

## Полезные команды

```bash
# Логи бэкенда в реальном времени
docker logs -f nocode-backend

# Подключиться к БД
docker exec -it lesovik-postgres psql -U app_user -d nocode_db

# Проверить очередь Celery
docker exec nocode-worker celery -A app.worker.celery_app inspect active

# Применить миграции вручную
docker exec nocode-backend alembic upgrade head

# Откатить последнюю миграцию
docker exec nocode-backend alembic downgrade -1

# Сбросить БД (ОСТОРОЖНО — только dev!)
docker exec -it lesovik-postgres psql -U app_user -c "DROP SCHEMA public CASCADE; CREATE SCHEMA public;"
docker exec nocode-backend alembic upgrade head
```

## Структура проекта бэкенда

```
backend/
├── app/
│   ├── api/
│   │   └── v1/
│   │       ├── endpoints/     # FastAPI роутеры (по одному на домен)
│   │       ├── deps.py        # AuthDep, DbDep — dependency injection
│   │       └── router.py      # Главный роутер, подключает все endpoints
│   ├── core/
│   │   ├── config.py          # Настройки через pydantic-settings
│   │   ├── database.py        # AsyncSession + get_db
│   │   ├── security.py        # hash_password, verify_password
│   │   ├── locks.py           # EditLock (Redis-based)
│   │   └── rate_limit.py      # SlowAPI rate limiter
│   ├── models/                # SQLAlchemy ORM модели
│   │   ├── identity.py        # User, Role, UserRole
│   │   ├── catalog.py         # App, AppMember
│   │   ├── data.py            # Entity, Field, Record
│   │   └── ...
│   ├── schemas/               # Pydantic схемы (request/response)
│   ├── services/              # Бизнес-логика (один сервис на домен)
│   ├── worker/                # Celery задачи
│   └── main.py                # Точка входа FastAPI
├── alembic/                   # Миграции
├── tests/                     # Pytest тесты
└── pyproject.toml
```
