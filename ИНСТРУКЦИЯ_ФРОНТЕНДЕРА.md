## API-слой

### Клиент (`shared/api/client.ts`)

`apiClient` — Axios instance с базовым URL `/api/v1`. Автоматически:
- Добавляет `Authorization: Bearer {token}` к каждому запросу
- При `401` делает одну попытку refresh токена и повторяет запрос
- При повторном `401` редиректит на `/editor/signin`

### Структура API файлов

Каждый файл в `shared/api/` отвечает за один домен:

```
apps.ts       → App, AppMember, listApps, createApp, updateApp, deleteApp,
                publishApp, listAppMembers, addAppMember, removeAppMember
auth.ts       → login, logout, refreshToken, getCurrentUser, changePassword
entities.ts   → Entity, Field, FieldType, listEntities, createEntity,
                createField, updateField, deleteField
records.ts    → RecordRead, listRecords, createRecord, updateRecord, deleteRecord
users.ts      → User, listUsers, createUser, updateUser
views.ts      → ViewRead, PageRead, listPages, createPage, updatePage, deletePage
health.ts     → getHealthReady (DB + Redis статус)
auditlogs.ts  → AuditLogEntry, listAuditLogs
orgs.ts       → Org, listOrgs
...
```

### TanStack Query хуки (`shared/hooks/`)

Каждый домен имеет свой хук-файл. Паттерн:

```typescript
// Чтение данных
export function useApps() {
  return useQuery({ queryKey: ["apps"], queryFn: listApps });
}

// Мутация с инвалидацией кэша
export function useCreateApp() {
  const qc = useQueryClient();
  return useMutation({
    mutationFn: (body: AppCreate) => createApp(body),
    onSuccess: () => { void qc.invalidateQueries({ queryKey: ["apps"] }); },
  });
}
```

**Все хуки инвалидируют нужный queryKey в `onSuccess`** — данные обновляются автоматически.

## Соглашения по коду

### Tailwind + cn()

Для условных классов используется `cn()` (обёртка над `clsx` + `tailwind-merge`):

```typescript
import { cn } from "@/shared/lib/utils"; // или из clsx/tailwind-merge напрямую

className={cn(
  "base-class px-4 py-2",
  isActive && "bg-cta text-white",
  disabled && "opacity-50 cursor-not-allowed",
)}
```

### Цветовые токены (CSS переменные)

Не используйте хардкод цвета! Все цвета — через CSS переменные темы:

```css
text-primary        /* основной текст */
text-cta            /* акцентный цвет (синий) */
bg-cardbg           /* фон карточек */
bg-mainbg           /* фон страницы */
border-white        /* разделители */
```

### SVG иконки

Иконки пишутся как inline React-компоненты. **Всегда задавайте `stroke` явно**, не через `stroke-current` (ненадёжно наследуется):

```typescript
function MyIcon() {
  const c = "#00205F";
  return (
    <svg viewBox="0 0 24 24" fill="none" className="w-6 h-6">
      <path d="..." stroke={c} strokeWidth="2" strokeLinecap="round"/>
    </svg>
  );
}
```

### Состояние страниц

Настройки представлений хранятся в `page.layout` (JSONB на бэкенде). Изменения:

```typescript
function patchLayout(partial: Record<string, unknown>) {
  const next = { ...layoutRef.current, ...partial };
  setLayout(next);                          // локальный стейт — мгновенно
  updatePageMutation.mutate({ pageId, body: { layout: next } }); // сохранение в БД
}
```

### Правило именования файлов

- Страницы: `PascalCasePage.tsx`
- Компоненты: `PascalCase.tsx`
- Хуки: `useCamelCase.ts`
- API: `camelCase.ts`

## MSW — работа с моками

Файл `src/mocks/handlers.ts` — полноценный in-memory бэкенд для разработки без Docker.

### Добавить новый мок-эндпоинт

```typescript
// в src/mocks/handlers.ts
import { http, HttpResponse } from "msw";

export const handlers = [
  // ...существующие
  http.get(`${API}/my-feature`, () => {
    return HttpResponse.json({ items: [] });
  }),
  http.post(`${API}/my-feature`, async ({ request }) => {
    const body = await request.json();
    return HttpResponse.json({ id: crypto.randomUUID(), ...body }, { status: 201 });
  }),
];
```

### Включить мок-режим

```bash
npm run dev:mock
```

// Мобильный

## Частые задачи

### Добавить новый API-метод

1. Добавить функцию в `shared/api/{domain}.ts`
2. Добавить хук в `shared/hooks/use{Domain}.ts`
3. Использовать хук в компоненте

### Добавить новую страницу

1. Создать `src/pages/MyPage.tsx`
2. Добавить роут в `src/editor/EditorApp.tsx`:
   ```typescript
   <Route path="/my-page" element={<RequireAuth><ScaleToFit><MyPage /></ScaleToFit></RequireAuth>} />
   ```
3. При необходимости добавить в `IconRail.tsx` и `MODULE_ROUTES`

### Добавить новую модалку

В `src/components/modals/Modals.tsx` добавить компонент и экспортировать его. Все модалки рендерятся через `<Overlay>`.

### Добавить новый тип поля

В `src/shared/api/entities.ts` расширить `FieldType`, добавить иконку в `DatabasePage.tsx`.

## Переменные окружения

Файл `.env` в корне `frontend/` (не в репозитории):

```env
VITE_API_URL=http://localhost:8000    # URL бэкенда (для dev)
VITE_USE_MOCKS=true                   # включить MSW моки (для dev:mock)
```

В продакшне Nginx проксирует `/api/v1` на бэкенд — переменные не нужны.

## Деплой

Сборка → Nginx → Docker. Команда на сервере:

```bash
python .deploy/ssh_run.py "cd /opt/lesovik/repo && git pull origin main && docker compose -f infra/docker-compose.server.yml --env-file infra/.env up -d --build frontend"
```

Vite собирает два бандла:
- `/editor` — редактор (основное приложение)
- `/runtime` — рантайм (опубликованные приложения пользователей)
