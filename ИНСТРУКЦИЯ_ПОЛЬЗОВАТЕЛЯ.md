# Инструкция пользователя — платформа Lesovik

---

## 1. Начало работы

### Вход в систему
Откройте `/editor` в браузере. Введите email и пароль. Если забыли пароль — нажмите «Забыли пароль?» для сброса на email.

### Выбор проекта
После входа открывается главная страница со всеми вашими приложениями. Нажмите на карточку — откроется предпросмотр справа. Кнопка «Открыть» запустит редактор.

### Создание нового проекта
Нажмите «+ Создать проект» на главной. Введите:
- Название
- Slug (только латиница, например `my-app`)
- Описание (опционально)
- Иконку и цвет (опционально)

После создания проект откроется в редакторе.

### Поделиться проектом
Нажмите кнопку «+» (добавить пользователя) в верхней панели навигации. В открывшемся окне:
- Введите email коллеги и нажмите **Enter** — он будет добавлен с ролью «Редактор»
- Кнопка «Ссылка» скопирует публичную ссылку на приложение

## 3. Конструктор — редактор представлений

### Страницы (левая панель)
- Список страниц приложения отображается слева
- Кнопка **«+»** — создать новую страницу
- Системные страницы (`_Detail`, `_Form`, `_Inline`) создаются **автоматически** при создании таблицы-сущности

### Тип представления
Выберите как будут отображаться данные:

| Тип | Описание |
|-----|----------|
| **Таблица** | Строки и колонки, как в Excel |
| **Галерея** | Сетка карточек с изображениями |
| **Карточка** | Карточки с аватаром, заголовком, подзаголовком |
| **Форма** | Форма ввода данных |
| **Детали** | Детальная карточка одной записи |
| **Дашборд** | Набор виджетов и графиков |
| **Диаграмма** | Столбцы / линейный график |
| **Календарь** | События по датам |
| **Колода** | Карточки для перелистывания |
| **Kanban** | Колонки по статусу |
| **Карта** | Геолокация точек |
| **Онбординг** | Пошаговый сценарий для новых пользователей |

### База данных
Выберите таблицу-источник для страницы. После выбора появятся настройки: сортировка, группировка, фильтрация колонок.

### Параметры представления

- **Сортировка** — по одному или нескольким полям (по возрастанию / убыванию)
- **Группировка** — сгруппировать строки по значению поля
- **Порядок колонок** — скрыть ненужные поля или задать порядок вручную
- **Ширина колонки** — Узкая / Средняя / Широкая
- **Быстрое редактирование** — правка прямо в таблице без открытия формы

### Стиль страницы (только для типа «Карточка»)

| Стиль | Внешний вид |
|-------|-------------|
| **Карточки** | Аватар + заголовок + изображение |
| **Список** | Горизонтальная строка с аватаром и текстом |
| **Сетка** | Квадратная плитка, фото сверху |
| **Полная** | Широкая карточка без аватара |

### Блоки страницы
Кнопка **«+ Добавить блок»** добавляет дополнительные элементы:

`Форма` `Таблица` `Кнопка` `Текст` `Метрика` `KPI` `График` `Kanban` `Календарь` `Разделитель` `Фрейм`

### Вкладки редактора

| Вкладка | Что настраивается |
|---------|-------------------|
| **Представления** | Блоки, источник данных, тип, параметры |
| **Правила формирования** | Условная логика: скрыть/показать поля, обязательность, вычисления |
| **Дизайн** | Шрифты, цветовая схема, акцент, скругления кнопок |

### Превью (правая панель)
- Живой предпросмотр приложения в реальном времени
- **Смартфон** — рамка 380×800, скругление 40px
- **Десктоп** — рамка 540×500, скругление 14px (плавная анимация при переключении)
- Кнопка **«Открыть»** запускает приложение в отдельном окне

## 4. База данных — управление структурой

### Создание таблиц
Раздел «Данные» → кнопка **«+ Добавить таблицу»**. Введите название — таблица сразу появится в списке слева.

### Типы полей
Нажмите **«+ Добавить поле»** в заголовке таблицы. Доступные типы:

| Группа | Типы |
|--------|------|
| Текст | Текст, Длинный текст, Rich Text |
| Числа | Число, Дробное |
| Логика | Да/Нет |
| Дата и время | Дата, Дата+время, Время |
| Выбор | Список (select), Мультисписок |
| Медиа | Файл, Изображение |
| Связи | Связь (relation), Lookup |
| Вычисления | Формула |
| Контакты | URL, Email, Телефон |
| Прочее | JSON |

### Фильтрация и сортировка
- Кнопка **«Фильтр»** — несколько условий по любому полю: `содержит`, `равно`, `начинается с`, `больше/меньше`, `пустое`
- Кнопка **«Сортировка»** — по одному или нескольким полям

### Вид: Таблица / Сетка
Кнопка **«Сетка ▼»** в верхней панели переключает между табличным и сеточным отображением.

### История изменений схемы
«История схемы» — журнал всех структурных изменений: добавление/удаление полей, переименования.

## 5. Безопасность

| Раздел | Описание |
|--------|----------|
| **Вход в систему** | Способы аутентификации: логин/пароль, SSO, OAuth |
| **Права полей (ABAC)** | Кто может видеть и редактировать каждое поле по роли |
| **Защитные фильтры** | Каждый пользователь видит только «свои» записи |
| **Аутентификация** | Настройка сессий и токенов |
| **Ограничение по домену** | Разрешить вход только с корпоративного домена (`@company.ru`) |
| **Журнал аудита** | История всех действий пользователей с уровнями Info / Warn / Error |

### Роли пользователей
- **Платформ-Администратор** — полный доступ ко всему
- **Администратор приложения** — управление конкретным приложением
- **Редактор** — создание и изменение данных
- **Просмотр** — только чтение

## 8. Автоматизация (боты и сценарии)

### Создание бота
Раздел «Автоматизация» → **«+ Новый бот»**. Задайте триггер (событие) и последовательность действий.

### Доступные действия в сценарии

| Действие | Описание |
|----------|----------|
| **Добавить строку** | Создать новую запись в таблице |
| **Удалить строку** | Удалить запись по условию |
| **Изменить значение** | Обновить поле у существующей записи |
| **Отправить email** | Письмо на произвольный адрес |
| **Уведомление** | Push-уведомление пользователю |
| **SMS** | Текстовое сообщение |
| **Webhook** | HTTP-запрос на внешний URL |
| **Выполнить скрипт** | Произвольный код |
| **Задание ИИ** | Обработка данных с помощью ИИ |

## 9. Что можно реализовать

### CRM / учёт клиентов
Таблица «Клиенты» с полями Имя, Email, Телефон, Статус. Представление-таблица + детальная карточка. Форма добавления. Связь с таблицей «Сделки». Бот отправляет email при создании нового клиента.

### Склад и инвентаризация
Таблицы «Товары» и «Движение». Остаток — поле Lookup или Формула. Фильтр по категории. Уведомление (бот) при остатке ниже нормы.

### Планировщик задач / Kanban
Таблица «Задачи» с полями Название, Ответственный, Дата, Статус (Список). Представление Kanban по полю Статус. Форма создания задачи. Уведомление при смене статуса.

### Приём заявок и заказов
Публичная форма приёма заявок (без входа). Таблица заявок для менеджера. Бот отправляет email-подтверждение при создании заявки. Статусы обработки.

### Корпоративный портал для сотрудников
Разные роли: руководитель видит всё, сотрудник только свои записи (защитные фильтры). Онбординг-экран для новых сотрудников. Дашборд с KPI и метриками.

### Дашборд / аналитика
Страница с блоками Метрика (одно число), KPI (с динамикой), График (столбцы/линия) на основе данных из таблиц. Переключатель Смартфон / Десктоп.
